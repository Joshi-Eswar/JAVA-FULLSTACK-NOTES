Stored Procedure

STORED PROCEDURE

  • A stored procedure is a prepared SQL code that you can save, so the code can be reused over and over again.
  • So if you have an SQL query that you writeover and over again, save it as a stored procedure, and then just call it to execute it.
  • You can also pass parameters to a stored procedure, so that the stored procedure can act based on the parameter value(s) that is passed.

SYNTAX FOR CREATING STORED PROCEDURE

//DELIMITER

CREATE PROCEDURE procedure name (parameter list)
BEGIN
...code goes here...
END //

DELIMITER ;

EXECUTING STORED PROCEDURE

CALLstored_procedure_name(parameters);

DROPPING THE STORED PROCEDURE

Drop procedure procedure_name;

We can pass the value to the procedure.
Parameters are used to pass values to the Procedure
There are 3 different types of parameters
1.IN
2.OUT
3.IN OUT

Example for IN
delimiter //
create procedure SalaryIncrementation(in designation varchar(15),in sal int)
begin
update empdetails set salary =salary+sal where designation=designation;
end//

delimiter ;

call SalaryIncrementation('Manager',1000);

Example for OUT
delimiter //
create procedure HighestAmountPaid(out Highest int)
begin
select max(amountpaid) into Highest from orders where pymtmode='Cash';
End//

delimiter ;

call HighestAmountPaid(@Highest);
select @Highest;
Example for INOUTset @sum=0;
delimiter ??
create procedure demo(inout sumint,in inc int)
begin
set sum=sum+inc;
End//

delimiter ;

call demo(@sum,1);
select @sum;
