Triggers

Triggers

  • Triggers in SQL is kind of stored procedures used to create a response to a specific action performed on the table such as Insert, Update or Delete. You can invoke triggers explicitly on the table in the database.
  • Action and Event are two main components of SQL triggers when certain actions are performed the event occurs in response to that action.

Syntax:
delimiter //
CREATE TRIGGER name
{BEFORE|AFTER} (event)}
ON table_name
FOR EACH ROW
BEGIN
...code goes here...
END//
delimiter ;


DROPPING THE TRIGGER

To delete or destroy a trigger, use a DROP TRIGGER statement. You must specify the schema name if the trigger is not in the default (current)schema :

Syntax:
Drop Trigger Trigger_name;

Create a trigger to display the aggregate, of the students automatically, while inserting a new student record.
Solution: delimiter $$
create trigger student
before insert on student_table
for each row
begin
set new.avg=(new.sem1+new.
sem2+new.sem3+new.sem4+new.sem5+new.sem6+new.
sem7+new.sem8)/8;
end
$$
delimiter ;


Create a trigger to calculate the no of days it takes to deliver the item to the customer, from the day of payment.
NOTE: Create a new column delivered_in_days.
Solution:delimiter $$
create trigger days_delivered
BEFORE insert on orders_trigger
for each row
begin
set new.delivered_in_days=(NEW.DELIVEREDDATE-new.PYMTDATE);End
$$
delimiter ;


Create a trigger to automatically set the managerid field to null when the details of a manager is removed from empdetails table.
Solution: delimiter $$
create trigger EMP_RESIGN
AFTER DELETE on empdetails
for each row
UPDATE retailoutlet
SET MANAGERID=null
WHERE MANAGERID=OLD.EMPID;
$$
delimiter;
