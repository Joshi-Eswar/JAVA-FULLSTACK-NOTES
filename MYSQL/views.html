VIEWS

views

A View can be defined as a virtual table that contains rows and columns with fields from one or more table.

  • A view is an alternative way to present a table (or tables).
  • You might think of a view as a "virtual" table.
  • A view is (usually) defined in terms of one or more tables.
  • When you create a view, you are not storing more data. You are instead creating a different way of looking at existing data.
  • A view is a useful way to give a name to a complex query that you may have to use repeatedly.


CREATING VIEW



A view can be created using the CREATE VIEW statement. We can create a view from a single table or multiple tables.

Syntax:
CREATE VIEW view_name
AS
SELECT column1,column2....
FROM table_name
WHERE condition;

Simple views and complex views

Feature simple complex
no of tables one more
function no yes
DML operation yes no

DELETING VIEW

A View can be deleted using the Drop View statement.
Syntax:
DROP VIEW view_name;
Example:
DROP VIEW Marks View

USES OF VIEWS:
  • To restrict data access
  • To make complex queries easy
  • To provide data independence
  • To present different views of the same data

Create a view named customer_information to display the customer id, customer name, email id and address of the customers.
Solution: create or replace view customer_information
AS
select custid,custname,emailid,address
from customer;
To fetch: select * from custinformation;
To delete:Drop view custinformation;
