INDEX

  • An index is used to reduce the amount of time it takes to perform certain operations.
  • Indexes are special lookup tables.
  • It is used to retrieve data from the database very fast.•An Index is used to speed up select queries and where clauses.

How are indexes created?

  • automatically: A unique index is created automatically when we define a primary key or a unique constraint
  • manually: users can create a non-unique index on column to speed up access to the rows

CREATE INDEX STATEMENT:

It is used to create an index on a table. It allows duplicate values
Syntax:
CREATE INDEX index_name
ON table_name(column1,column2....);

Example:
WAQ to create an index named "index_quantity" on qtyavailable column present inretailstocktable.
Solution: create index index_quantity
on retailstock(qtyavailable);

UNIQUE INDEX STATEMENT:

It is used to create a unique index on a table. It does not allow duplicate value.
Syntax:
CREATE UNIQUE INDEX index_name
ON table_name(column1,column2...);
Example:
Create an unique index empdesignation on the empname and designation column sof empdetails
Solution: create unique index empdesignation
on
empdetails(designation,empname);

DROP INDEX STATEMENT:

It is used to delete an index in a table

Syntax:
DROP INDEX index_name on table_name;

Example:
Drop index index_quantity on retailstock;
Drop index empdesignation on empdetails;

When to create an index?

➢A column contains a wide range of values

When not to create an index?

➢the table is small
➢columns are not often used as a condition in the query
➢the table is updated frequently
In order to view the indexes present on a table we can say
SHOW INDEXES FROM table_name
