Introduction

Introduction



DATA: Data is small volume of facts and figures
DATABASE: Database is large volume of facts and figures

Types of Data

1.Structured
2.Unstructured
3.Semi-Structured

Structured:
Structured data is data which have a predefined data model
Structured data conforms to a tabular format with the relationship between the different rows and columns.

Semi-Structured:
Semi-structured data is a form of structured data that does not conform with the formal structure of data models

Unstructured:
Unstructured data is information that either does not have a predefined data model or is not organized in a predefined manner.

Types of Databases


1.Centralized Database
2.Distributed Database
3.NoSQL Database
4.Cloud Database
5.Relational Database
6.Network Database
7.Hierarchical Database

➢Centralized Database: It is the type of database that stores data at a centralized database system. It comforts the users to access the stored data from different locations through several applications. These applications contain the authentication process to let users access data securely.
➢Distributed Database: Unlike a centralized database system, in distributed systems, data is distributed among different database systems of an organization. These database systems are connected via communication links. Such links help the end-users to access the data easily.
➢NoSQL Database: Non-SQL/Not Only SQL is a type of database that is used for storing a wide range of data sets. It is not a relational database as it stores data not only in tabular form but in several different ways. It came into existence when the demand for building modern applications increased.
➢Cloud Database: Cloud database facilitates you to store, manage, and retrieve their structured, unstructured data via a cloud platform. This data is accessible over the Internet.
➢Relational Database: This database is based on the relational data model, which stores data in the form of rows(tuple) and columns(attributes), and together forms a table(relation). Four commonly known properties of a relational model known as ACID properties.
➢Network Database:In this model data is organized more like a graph and are allowed to have more than one parent node.
➢Hierarchical Database:It is the type of database that stores data in the form of parent-childrenrelationship nodes.
