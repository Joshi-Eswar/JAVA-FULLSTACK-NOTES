KEYWORDS AS OPERATORS


DISTINCT



The DISTINCT keyword is used to remove duplicate records from the table and fetch only the uniquerecords.
The DISTINCT keyword is not dependent on the data type of the column.

Note:
unique should be used while creating the table.
Distinct should be used while retrieving.

Syntax:
SELECT DISTINCT expression
FROM tables
[WHERE conditions];

Displayall the unique type of customer.
Solution:Select distinct custtype
From customer;

Display the unique items from the item table.

Solution:Select distinct itemtype
From item;

Display unique types of customers and their address.
Solution:Select distinct custtype, address
From customer;

BETWEEN AND


The MYSQL BETWEEN condition specifies how to retrieve values from an expression within a specific range.
Syntax:
expression BETWEEN value1 AND value2;

It is used to select a value which lies between the given range. (value between Upper limit and Lower limit).

IN and NOT IN


The MYSQL IN condition is used to reduce the use of multiple OR conditions.
Whenever comparison has to be done with respect to a set of values, we have to use the operator IN.
Syntax:
expression IN(value1,value2,.....valuen);

LIKE

In MYSQL, LIKE condition is used to perform pattern matching to find the correct result.
➢%(modulus) matches 0 or more characters.
➢_(underscore) it matches exactly one character.

Regex

➢^ -starting with
➢$ -ending with
➢[ ] -matches any single character within the specified range inside

ESCAPE

Escape it is a special keyword which converts the metadata into ordinary character.
Example:
Consider a relation employees.
Display employee details whose email_id contains underscore in it.
Solution:select *from employees
where email_id like '%?_%' escape '?';
