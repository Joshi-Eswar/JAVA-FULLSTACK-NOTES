JOINS

JOINS

  • It is used to retrieve data from multiple tables.
  • It is performed whenever you need to fetch records from two or more tables.
  • Join clause is used to combine or compare rows from two or more tables, based on related column between them.

Types of Joins:

➢INNERJOIN (SIMPLE JOIN) OR EQUI JOIN
➢NATURAL JOIN
➢OUTER JOIN
  • LEFT OUTER JOIN (LEFT JOIN)
  • RIGHT OUTER JOIN (RIGHT JOIN)
➢CROSS JOIN/CARTESIAN JOIN
➢SELF JOIN

INNER JOIN (SIMPLE JOIN) OR EQUI JOIN

The INNER JOIN is used to returntherows from multiple tables where the join condition is satisfied.
It is most common type of join.
Inner join is applicable only if there is commoncolumn, ifnot then inner join is invalid.

Syntax of inner join:
Select */exp
From table1 inner join table2
On table1.column=table2.column;

Syntax of equi join:
Select */exp
From table1,table2
Where table1.column=table2.column;

NATURAL JOIN

The tables should not have one or more pairs of identical named columns.
NOTE:Do notuse “on” clause in natural join.
Syntax:
Select *
From tab1 natural join tab2;

OUTER JOIN

Outer join gives the non-matching records along with matching records.

LEFT OUTER JOIN

•The LEFT OUTER JOIN returns all rows from theleft handtable and only those rows from the other table where the join condition is fulfilled.
•This will display all matching records and the records which are in left hand side table those that are not in right hand side table.

Syntax:
SELECT*/columns
FROM table1 LEFT OUTER JOIN table2 ON table1.column=table2.column;

RIGHT OUTER JOIN

•The RIGHT OUTER JOIN returns all rows from right hand table and only those rows the other table where the join condition is fulfilled.
•This will display all matching records and the records which are in right hand side table those that are not in left hand side table.
Syntax:
SELECT*/columns
FROM table1 RIGHT OUTER JOIN table2 ON table1.column=table2.column;

FULL OUTER JOIN

•This will display all matching records and the non-matching records from both tables.
•This is combination of left outer join and right outer join.
Syntax:
SELECT */columns
FROM table1 LEFT OUTER JOIN table2 ON table1.column=table2.column
UNION
SELECT */columns
FROM table1 RIGHT OUTER JOIN table2 ON table1.column=table2.column;

CROSS JOIN/CARTESIAN JOIN

•It is cross product of 2 tables.
•Works on the principle that each and every record of one table matches with all records of other table.
•Cartesian joins results in valid results and also invalid results.(matchindand unmatching)
•Always formed without any joining condition.

Syntax of cross join:
SELECT */exp
FROM table1 cross join table2;

Syntax of cartesian join:
SELECT */exp
FROM table1,table2;
The output of Cartesian is in the format of cross product hence name cross join.


SELF JOIN

•A self-join is a query in which a table is joined (compared) to itself.
•Self-joins are used to compare values in a column with other values in the same column in the same table.
•At run time the tables will be created with 2 copies

Display all the details of purchases done by customers,and also display all the customer’s details even if they have not done any purchase.
Solution: select *
from purchasebill p right join customer c
on p.custid=c.custid

Solution:select empname,custname
